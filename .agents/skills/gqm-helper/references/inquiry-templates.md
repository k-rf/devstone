# 探索を深めるための問いかけテンプレート

GQM（GQMD）の探索中、エージェントは以下の問いかけを適宜行い、ユーザーの思考を刺激する。
これらの問いかけは、ユーザー自身が気づいていない本質的な課題感や、メトリクスの隠れた副作用を引き出すために役立つ。

| フェーズ           | 問いかけの例                                                                                           | 意図                                         |
| :----------------- | :----------------------------------------------------------------------------------------------------- | :------------------------------------------- |
| **Goalの深掘り**   | 「この測定を始めようと思った、直接のきっかけ（課題感）は何ですか？」                                   | 表面的な目標の裏にある真の動機を突き止める   |
| **Goalの深掘り**   | 「もしこの目標が完璧に達成されたら、誰が一番喜びますか？ どんな変化が起きていますか？」                | 視点と理想の状態を具体化する                 |
| **Questionの探索** | 「この目標に向かう中で、今一番ボトルネックになっていそうな懸念や『不確実なこと』は何ですか？」         | 優先度の高い問いを絞り込む                   |
| **Questionの探索** | 「目標が達成できているかどうかを、他人に説明するとしたら、どんな事実を提示しますか？」                 | 判断基準を明確にする                         |
| **Metricの検証**   | 「もしこのメトリクス（数値）が改善したのに、状況が全く良くならないとしたら、何が原因だと思いますか？」 | メトリクスのハックや形骸化のリスクを洗い出す |
| **Metricの検証**   | 「このデータを取るために、開発者がどれくらいの負荷（手動記録など）を負うことになりますか？」           | 測定コストと持続可能性のトレードオフを考える |
| **Dataの定義**     | 「その指標を計算するために必要な最小限の『生データ』は、どのツールのどこに記録されていますか？」       | 実装の実現可能性と取得ソースを明確にする     |
