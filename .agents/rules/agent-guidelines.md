# Agent Guidelines

> [!NOTE] このファイルの役割
> エージェント自身のカスタマイズ（スキル・ルール）に関する設計方針や、開発における対話のあり方を定義しています。

## 1. スキルとルールの定義と役割

エージェントを構成・カスタマイズするにあたり、「スキル」と「ルール」は明確に役割を分離して定義します。

### スキル（Skill）: ふるまいと手順

- **定義**: エージェントが実行する具体的な手順や、振る舞いを記述したものです。
- **性質**: 「実行する」ものであり、目的を達成するための道具や手段として機能します。

### ルール（Rule）: 作法と境界

- **定義**: エージェントがやってはいけないこと（禁止事項）や、推奨することを記述したものです。
- **性質**: 「守る」ものであり、エージェントの行動を制御し、安全に開発を進めるための境界線として機能します。
