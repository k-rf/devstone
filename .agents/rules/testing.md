# Testing Strategy

> [!NOTE] このファイルの役割
> プロジェクトにおけるテストの方針、分類、および実装方法を記載しています。

## 1. テストの分類

本プロジェクトでは、ピラミッド型のテスト構成を意識し、低いレイヤほど厚くテストを行います。

| 種類                 | 対象               | ツール | 目的                                           |
| :------------------- | :----------------- | :----- | :--------------------------------------------- |
| **Unit Test**        | `core/domain`      | Vitest | 純粋なロジックの正当性を検証。                 |
| **Integration Test** | `core/application` | Vitest | Port をモックし、Service の流れを検証。        |
| **Adapter Test**     | `adapter/outbound` | MSW    | 外部 API との通信（変換ロジック等）を検証。    |
| **E2E Test**         | `index.ts` (Entry) | Vitest | 全レイヤを結合し、実際の挙動を検証（最小限）。 |

## 2. 依存関係の制御

Effect-TS の `Layer` を活用し、テスト時の副作用を制御します。

- **Layer Overriding**: テスト時には `core/application` が依存する Port を、インメモリの実装や Mock 実装（`TestLayer`）に差し替えます。
- **No Real I/O**: ユニットテスト・統合テストにおいて、実際の外部 API や KV に接続してはいけません。

## 3. モックの方針

- **Internal Mocks**: `core/port` を実装したシンプルなクラス、または `Effect.succeed` を返すオブジェクトを使用します。
- **External Mocks**: HTTP 通信が伴う場合は `MSW` を使用し、ネットワーク層でのモックを徹底します。これにより、Adapter 層の変換ロジックも同時にテスト可能になります。

## 4. テストの実装指針

- **In-source Testing**: ユニットテスト（特に `core/domain` などの独立したロジック）は、
  ソースファイル内に `if (import.meta.vitest)` ブロックを用いて記述することを推奨します。
  - テスト対象と同一ファイル内に記述することで、カプセル化されたロジックの検証を容易にし、コードとテストの距離を近づけます。
  - ただし、Application 層（Service）などの統合テストにおいて、多くのモックや `Layer` のセットアップが必要で、ソースコードの可読性を損なう（ノイズが大きすぎる）場合は、
    `.spec.ts` ファイルとして分離してください。
  - インソーステスティングにおいて、ファイル名や関数名を繰り返すだけのトップレベルの `describe` は不要です。
- **Predictability**: ランダムな値の使用を避け、入力に対して期待される出力が常に一定であることを保証します。
- **Descriptive Titles**: テストケースの名前は、ビジネス要件を反映した文章として記述します。
  - 悪い例: `test('成功する', ...)`
  - 良い例: `test('Notionでタスクが更新されたとき、新しいタイマーを開始する', ...)`
- **Data-Driven**: 境界値テストなど、パターンが多い場合は `test.each` を活用して網羅性を高めます。

## 5. 信頼性はカバレッジ100%の先にある

- **カバレッジ100%は前提**: カバレッジが100%でないということは、確実に検証されていないコードが存在することを意味します。まずは網羅することを目指します。
- **真の信頼性の探求**: カバレッジ100%だからといって、本当に検証したいことを検証しきれているとは限りません。数値そのものを追うのではなく、「壊れたら必ず失敗する」意味のあるテストを書くことを重視します。
- 特に、複雑なパターンマッチング（`Match`）やエラー分岐は重点的にテストしてください。
